# hakhatz.dev

個人サイト

## 方針

- 静的ファイルで構築
- 装飾は最小限に留め、シンプルさによる資格負荷の低減の面から読みやすさを追求する
- HTMLとブラウザのデフォルトに主軸を置き、補助としてCSS/JSを使用する
- Google Lighthouseを活用してパフォーマンスの改善に努める

## Gitコミットプレフィックス

1. `feat:`
    - Feature
    - 新機能の追加
    - アプリケーションに新しい機能、新しい画面、新しいAPIなどを実装
2. `fix:`
    - Bug Fix
    - バグや不具合の修正
    - 既存のバグ、クラッシュ、表示崩れ、誤字脱字、仕様と異なる挙動などを修正
3. `refactor:`
    - Refactoring
    - コードのリファクタリング
    - バグ修正や新機能の追加を行わず、プログラムの挙動は変えないまま、コードの可読性や保守性を向上させるために構造を整理
4. `chore:`
    - Chore
    - 雑用、環境構築、ビルド設定
    - 製品のコードそのものには直接関係しない変更に使用
    - スクリプト、サイトマップの生成、ビルドツールの設定、依存ライブラリのアップデート、不要ファイルの削除、CI/CDの設定変更
5. `content:`
    - Content
    - Webページやブログなどのコンテンツ（文章・メディア）の更新
    - プログラムのロジックやデザイン（CSS）の変更を伴わない、テキスト、お知らせ、記事内容、画像URLなどの差し替えや追加・修正
6. `docs:`
    - Documentation
    - ドキュメントの修正
    - README.md、内部向けのドキュメント、設計書、コード内のコメントなどの追加・変更・修正
7. `style:`
    - Style
    - コードスタイルの修正（挙動に影響しない変更）
    - インデントの修正、スペースの有無、セミコロンの付け忘れ、フォーマッタ（Linter）による自動整形など、プログラムの実行結果に影響を与えない見た目の変更
8. `test:`
    - Test
    - テストコードの追加・修正
    - 自動テスト（ユニットテストや統合テスト）の追加、欠けていたテストケースの補填、既存のテストコードの修正
9. `perf:`
    - Performance
    - パフォーマンスの向上
    - 処理速度の向上、メモリ使用量の削減、アルゴリズムやクエリの最適化など、パフォーマンス改善を目的とした変更
10. `build:`
    - Build
    - ビルドシステム、外部依存関係の変更
    - 外部依存関係の追加・削除（package.json, Cargo.toml, go.mod などの変更）、WebpackやCMakeなどのビルド構成に関する変更（choreよりビルド周りに特化）
11. `ci:`
    - Continuous Integration
    - CI/CD設定の変更
    - GitHub Actions、GitLab CI、CircleCIなどの設定ファイルやスクリプトの追加・変更・修正（choreよりCI/CD周りに特化）

## ディレクトリ構成

- `.git/hooks/`
    - `post-commit`: PowerShellから`post-commit`を実行するためのスクリプト
    - `post-commit.ps1`: `hakhatz-dev/scripts/generate-sitemap.py`を実行しサイトマップを生成する
- `public/`: 公開される静的ファイルのルート
    - `css/`: スタイルシート
    - `images/`: アイコンや画像
    - `js/`: JavaScript(現在(2026-06-06)は未使用)
- `scripts/`: 開発・保守用スクリプトを格納する
    - `escape-code.py`: テキストを実体参照に変換するスクリプト
    - `generate-sitemap.py`: サイトマップ(`public/sitemap.xml`)を自動生成するスクリプト
    - `serve.ps1`: Pythonの標準機能を利用して、ローカル環境で簡易的なWebサーバーを起動するスクリプト
    - `tree.py`: ディレクトリ構造をYAML形式で出力するスクリプト

## ローカル開発環境

本プロジェクトはWindowsで開発されています。確認する場合はWindows PowerShellを使用してください。

### 実行例

```PowerShell
.\scrips\serve.ps1
```
